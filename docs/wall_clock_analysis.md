# Wall-Clock Analysis

## Motivation

仅仅报告真实仿真次数的减少，并不能完整反映一个 surrogate-assisted optimization 方法的实际效率。因为代理模型训练、ensemble 推理、不确定性校准以及在线更新也都需要消耗时间。如果这些额外成本过高，那么即便仿真调用数量下降，总体 wall-clock time 也未必减少。对于 EDA 场景而言，真实工程价值更接近“总时间是否下降”，而不是单一统计量是否变好。因此，本项目需要单独进行 wall-clock analysis。

## Time Components

总时间由至少三部分组成。第一部分是高保真仿真时间，即真实电路仿真的累计运行时间。第二部分是模型训练与更新开销，包括初始训练、周期性重训以及必要的校准步骤。第三部分是模型推理与决策开销，包括候选点批量预测、风险计算和调度模块运行。必要时，还可以将数据读写和缓存操作单独记录，但主要结论应围绕前三类时间展开。

## Measurement Protocol

所有方法都在相同硬件条件、相同批量设置和相同仿真并发限制下运行。每次实验必须记录开始时间、结束时间以及各阶段子模块耗时。对于包含在线更新的策略，必须单独统计每次重训耗时和重训频率。对于不同随机种子，应汇报平均总时间和方差，而不是只给出一次运行结果。

## Interpretation

如果本文方法减少了仿真次数，但训练与推理开销大幅上升，则必须诚实报告这一点。理想情况是：虽然引入了代理模型与风险调度开销，但由于显著减少了高成本仿真，总 wall-clock time 依然下降。即便在某些设置下总时间优势不明显，只要方法在高风险场景中显著提高了收敛可靠性，也应在文中明确说明效率与可靠性之间的权衡关系。

## Expected Role

wall-clock analysis 的主要作用，是让论文不只停留在“研究上成立”，而进一步具有工程可信度。尤其是在模拟电路优化这一高度工程化的方向中，单纯减少仿真次数往往不足以说服审稿人或工业读者。加入总时间分析，可以使本文从方法论贡献延伸到实际应用价值，从而增强整体说服力。
