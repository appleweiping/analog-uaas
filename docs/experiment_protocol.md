# Experiment Protocol

## Objective

本实验协议的目的是确保整个研究过程在不同方法、不同实验组和不同阶段之间保持可重复、可追踪和可公平比较。由于本项目的核心结论不仅依赖平均性能，还依赖于 failure mode、风险调度和不确定性质量，因此实验协议必须在一开始就冻结关键设置，而不能在后期根据结果临时调整。所有主结果、消融实验和困难场景实验，都应严格遵守本协议，除非在文档中显式记录变更。

## Circuit and Environment

第一篇工作固定研究对象为单一拓扑的两级运放，固定工艺模型、供电条件和仿真环境，不进行跨拓扑比较。之所以做这一限制，是为了避免将问题复杂度分散到结构变化上，而是聚焦于代理模型在约束边界附近的可靠性问题。所有实验共享同一套网表模板、同一套性能指标定义以及同一约束集合。若后续加入困难场景或 OOD 实验，也必须在保持拓扑不变的前提下进行轻量扰动，例如改变负载、spec 或供电条件。

## Initial Dataset

所有方法共享同一个初始数据池。初始样本由统一采样策略产生，例如 Latin Hypercube Sampling 或均匀随机采样，样本数量由配置文件固定。初始数据池生成完成后，不得在不同 baseline 之间重新抽样，以避免因初始条件差异导致的性能偏差。若实验需要多随机种子重复，则每个随机种子对应一个固定的初始数据版本，所有方法在该种子下共享同一版本。

## Training / Validation / Test Split

代理模型训练使用统一的数据划分方式。验证集用于模型选择、不确定性校准和调参；测试集仅用于最终性能分析与 failure mode 统计，不得反复用于超参数选择。对于在线更新或主动学习新增的数据，必须保留其阶段标签，并在分析时明确区分“初始静态数据性能”和“闭环补点后性能”。

## Optimization Budget

每个优化任务都具有固定的真实仿真预算上限，预算定义为可调用高保真仿真的最大次数。不同方法必须在相同预算条件下比较，不得通过增加预算换取更高性能。对于模型训练和推理引入的额外开销，在 wall-clock 分析实验中单独统计。若实验使用批量候选生成，则每轮批量大小、候选池大小和重训频率都应在配置中显式固定。

## Baselines

主结果实验至少包含以下 baseline：随机搜索或 LHS+真实仿真方法，用于提供最朴素的黑盒优化参照；纯代理模型优化方法，即训练好模型后仅依赖预测推进搜索，用于验证后期失效现象；标准贝叶斯优化或常用采集函数方法，用于回应“方法是否只是 BO 变体”的质疑；有不确定性但不含边界风险或 utility 建模的简化方法，用于支撑关键消融。本文提出的方法作为完整版本与上述基线统一比较。

## Metrics

实验评价指标分为四类。第一类是优化效率指标，包括达到目标 spec 所需仿真次数、平均成功设计预算以及总墙钟时间。第二类是优化质量指标，包括最终设计性能、约束满足率和成功收敛比例。第三类是稳定性指标，包括后期轨迹震荡程度、误判事件频率以及在边界附近来回往返的次数。第四类是不确定性质量指标，包括误差—不确定性相关性、置信区间覆盖率和校准误差。只有四类指标同时成立，方法的“可靠优化”主张才算被支撑。

## Failure-Case Experiment

为了证明 failure mode 具有实际影响，必须设置专门实验考察模型在接近约束边界时的表现。该实验统计不同边界距离区间上的预测误差、可行性误判率以及优化后期误导事件，并与纯代理方法或无边界风险控制的方法进行对比。此类实验不以最终性能为唯一目标，而以揭示为什么现有方法会失败为主要目的。

## Ablation Study

消融实验必须围绕方法的关键新增部分展开，而不是任意删改模块。建议至少包含以下设置：去掉不确定性校准，仅使用原始 ensemble variance；去掉 utility-driven 采样，仅按 uncertainty 选点；去掉误判风险建模，仅保留简单边界距离项；关闭在线更新，仅使用静态训练模型。通过这些消融，验证每个模块对最终表现的独立贡献。

## Hard-Case Protocol

困难场景实验用于放大 baseline 的弱点。可选设置包括极限 phase margin 约束、低功耗与高 GBW 同时收紧、多个约束同时逼近边界等。所有困难场景必须在独立配置文件中显式定义，并在结果中与主实验分开汇报。困难场景的目标不是让方法在所有条件下都更强，而是在最关键、最容易出现后期失效的设置中，展示本文方法的必要性和优势。

## OOD and Wall-Clock Extensions

为增强完整性，可加入轻量 OOD 实验，例如改变负载、电源电压或目标 spec，观察模型与优化策略在分布轻微变化下的表现。同时引入总时间分析，将仿真时间、训练时间和推理时间统一计入 wall-clock 成本，以避免只报告仿真次数而忽视整体效率。

## Reproducibility

所有实验必须由配置文件驱动，所有随机种子必须固定且显式记录，所有结果图表必须可由脚本自动重现。主实验、消融实验、困难场景实验和扩展实验都应能通过统一入口脚本复现，并在结果目录中自动记录对应配置、日志和生成图表的时间戳。可复现性不是论文附属，而是本项目方法有效性的一部分。
