# Methodology

## Overview

本项目的方法核心并不是简单地“训练一个代理模型来替代仿真”，而是围绕代理模型在优化过程中的可靠性问题构建一个闭环系统。具体而言，系统的设计遵循如下逻辑：首先，用高保真仿真器在固定拓扑与工艺条件下生成结构化数据；其次，训练一个多输出代理模型以学习参数到性能指标的映射关系；然后，对模型的不确定性进行建模与校准，使其能够更真实地反映模型在不同区域的预测风险；接着，将这种不确定性与约束边界信息结合，构造可行性误判风险；最后，用风险与效用联合驱动候选点的选择和仿真调用，形成代理模型与真实仿真协同工作的优化闭环。

## Base Surrogate Modeling

在固定拓扑条件下，模拟电路设计问题可视为一个从设计参数向量到多目标性能向量的回归问题。基础代理模型采用多输出 MLP，其输入包括已归一化的设计参数，输出包括 gain、GBW、phase margin、power 等关键性能指标。为了获取 epistemic uncertainty，本项目采用 ensemble 策略，即训练多个随机初始化的独立模型，在预测时取其均值作为性能估计，取其分布方差作为不确定性近似。相比更复杂的贝叶斯神经网络，这种方法在工程实现上更稳定，训练与调参成本更低，也更容易为审稿人接受。

## Failure-Aware Perspective

传统代理优化方法默认代理模型在整个设计空间中大致可靠，但本项目认为这一假设在真实优化过程中往往不成立，特别是在接近设计约束边界时。比如某个候选点的 gain 和 GBW 很接近目标要求，相位裕度仅略高于下界，而功耗又逼近上限，此时代理模型的微小误差就可能改变该点是否可行的判断。因此，本项目从一开始就将“边界附近的误判风险”作为方法设计的核心对象。方法不是围绕“如何降低整体 MSE”展开，而是围绕“如何避免在高风险点上做出错误决策”展开。

## Uncertainty Estimation and Calibration

不确定性本身只有在能稳定对应真实误差时才有方法价值。因此，本项目不仅计算 ensemble variance，还引入后验校准机制，使预测方差与真实误差统计更加一致。校准可以采用轻量级后处理方式，例如 variance scaling、quantile-based calibration 或保序映射，其目标不是重新训练模型，而是在验证集上将“不确定性大小”与“预测风险水平”对齐。经过校准后，高不确定性区域应更显著地对应高预测误差区域，从而使后续的风险评估和仿真调度建立在可信信号之上。

## Feasibility Risk Modeling

在模拟电路优化中，真正会误导优化器的并不是一般意义上的误差，而是“误将不可行点判断为可行”的错误。因此，本项目不满足于直接使用预测方差，而是进一步定义面向约束的误判风险。该风险结合三个来源的信息：第一，代理模型对各项性能指标的预测均值；第二，模型输出的不确定性；第三，候选点到最近约束面的 margin，即边界距离。一个预测表现优异但紧贴约束边界、同时置信度不足的点，应被视为高误判风险点。在方法实现中，这一风险既可以采用显式概率近似，也可以用分解式评分进行近似建模，但核心原则始终不变：风险量必须服务于“是否应调用真实仿真”的决策。

## Utility-Driven Active Sampling

仅根据不确定性选择样本，容易把仿真预算浪费在与优化目标无关的区域。为此，本项目将主动采样从 uncertainty-driven 升级为 utility-driven。这里的 utility 反映的是一个候选点对当前优化任务的潜在价值，它同时考虑性能改进潜力、可行性概率、误判风险以及不确定性水平。这样一来，系统优先关注的是“既有可能改进目标、又存在决策不确定性”的关键点，而不是全空间中简单意义上的高方差点。该机制保证主动学习不再只是泛化导向的补点，而是直接服务于优化过程本身。

## Adaptive Simulation Scheduling

代理模型与真实仿真的协同通过一个调度模块实现。对于每一批候选点，系统首先输出其预测性能、不确定性、边界距离和风险评分，然后根据预设或学习到的调度规则，将候选点划分为不同处理方式。在低风险区域，系统可以直接依赖代理模型推进局部搜索，以节省仿真预算；在中等风险区域，可以先做保守筛选后再仿真少量代表点；在高风险区域，尤其是预测接近约束边界或最优解附近的点，系统会优先调用真实仿真以校正模型。仿真结果返回后，样本被加入数据池并触发定期重训或增量更新，从而形成闭环。

## Closed-Loop Optimization Procedure

整体优化流程如下。首先，在设计空间内进行初始采样并运行高保真仿真，得到起始数据集。随后训练代理模型并完成初步校准。每一轮优化中，系统先由候选生成器提出候选点，再由代理模型预测其性能及不确定性；接着根据约束定义计算边界距离，并结合不确定性构建误判风险；然后由 utility-driven 采样器决定哪些点值得仿真、哪些点可以暂时依赖代理模型；新仿真结果回流数据池后，系统更新模型并进入下一轮迭代。最终，系统在尽量减少仿真次数的同时，保持对高风险区域的谨慎处理。

## Methodological Positioning

本项目的方法定位非常明确：不是试图证明“代理模型可以彻底替代 SPICE”，而是证明“代理模型可以在识别自身局限的前提下有效管理 SPICE 调用”。这一区别决定了方法的研究价值。前者更容易受到工业场景质疑，因为模拟电路天然对边界和极端条件敏感；后者则更符合实际需求，也为后续更高层次的 design agent 系统奠定方法基础。第一篇工作的主要任务，是把“模型什么时候值得信”这件事讲透、做实并验证清楚。
