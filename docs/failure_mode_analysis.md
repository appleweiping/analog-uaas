# Failure Mode Analysis

## Motivation

本项目的研究起点并不是“代理模型不够准”，而是“代理模型在优化过程中会以一种对决策极其有害的方式出错”。具体而言，这种错误主要发生在设计约束边界附近，并且在优化后期最容易放大。也就是说，一个在整体测试集上看似还不错的代理模型，仍然可能在最关键的决策点上系统性误导优化器。因此，failure mode analysis 的目标不是对模型误差做泛泛统计，而是要证明：现有 surrogate-assisted optimization 的真正瓶颈在于边界区误判，而不是全局平均精度不足。

## Definition of Failure Mode

本文关注的 failure mode 可定义为：当候选点接近设计约束边界时，代理模型对可行性的误判概率显著升高，尤其是更容易将真实不可行点预测为可行点；这种误判在优化后期会被反复利用，从而导致搜索轨迹偏离真实可行区域，表现为后期震荡、无效仿真增加以及收敛不稳定。这个定义包含三个关键要素。第一，它是边界相关的，而不是全局均匀分布的误差现象。第二，它是决策相关的，因为真正的问题不只是数值误差，而是可行性判断错误。第三，它是动态相关的，因为误判会在闭环优化中累积和放大。

## Analysis Questions

本分析文档围绕以下问题展开。第一，模型误差是否随着边界距离减小而系统性增大。第二，在边界附近，模型是否存在误差偏向，例如偏向将点判为可行。第三，这种误判是否足以改变优化轨迹，即是否导致 surrogate-only 或缺少风险控制的方法在后期出现反复试探和震荡。第四，不同方法是否在这一 failure mode 上表现出系统差异。

## Boundary Distance

边界距离定义为一个样本距离最近约束面的 margin。若样本的某项性能指标刚好逼近某个约束下界或上界，则其边界距离接近零。边界距离越小，说明样本越处于“少量预测误差即可改变可行性判断”的敏感区域。本项目在分析中会将样本按边界距离分桶，并比较不同桶中的误差分布、误判率和风险行为。

## Required Analyses

第一项分析是误差—边界距离关系图。通过比较不同边界距离区间上的绝对误差或归一化误差，验证模型在 boundary 区域是否表现出更高风险。第二项分析是可行性误判率曲线，特别关注 false feasible 类型错误，即预测满足约束但真实不满足约束的事件。第三项分析是误差方向性分析，考察在边界附近模型是否存在系统偏移，例如倾向于高估 phase margin 或低估 power。第四项分析是优化轨迹分析，比较不同方法在接近最优解时的轨迹行为，观察是否出现围绕边界的往返、停滞和反复仿真。

## Experimental Role

failure mode analysis 在整篇论文中的作用不是附属实验，而是方法存在必要性的证据。只有当这一分析清楚地表明：边界区误判真实存在、可测量、且会在优化过程中造成严重后果，后续的 uncertainty calibration、misclassification risk 和 simulation scheduling 才显得必要。否则，方法很容易被理解为只是对已有 surrogate optimization 稍作增强。

## Expected Findings

我们预期会观察到以下现象。第一，随着边界距离减小，预测误差和误判率明显上升。第二，纯代理优化方法在后期更容易反复进入预测上看似可行但真实不可行的区域。第三，仅用不确定性但不显式处理边界风险的方法虽然有改进，但仍无法稳定抑制边界误导。第四，引入风险感知调度后，系统会在关键区域增加真实仿真，从而显著减少后期误判累积。

## Output Artifacts

本分析最终应生成至少以下结果：误差—边界距离图、误判率—边界距离图、若干失败样本案例表，以及至少一张优化轨迹对比图。所有图表必须能由标准化脚本自动生成，并与具体配置文件、数据版本和模型版本一一对应。
