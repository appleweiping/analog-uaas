# Paper Figures Plan

## Purpose

本文件用于提前规划论文所需核心图表，使代码结构、实验脚本和论文写作从一开始保持一致。项目中的每张关键图都不应是“最后想起来再补”的结果，而应在实验设计阶段就明确其科学作用、输入数据来源以及对应的生成脚本。这样做的目的，是保证整篇论文围绕同一条问题—方法—证据链展开，而不是实验很多但图表分散。

## Figure 1: Framework Overview

第一张图应为系统框架图，展示从候选生成、代理预测、不确定性估计、风险评估、仿真调度到数据回流和模型更新的完整闭环。该图用于引言和方法总览，目标是帮助读者快速把握本文不是简单 surrogate regression，而是一个 reliability-driven model–simulation collaboration system。对应输入来自方法逻辑和模块结构，不依赖实验结果。

## Figure 2: Failure Mode Visualization

第二张图是 failure mode 核心证据图，可设计为误差—边界距离关系图或误判率—边界距离关系图。该图应清楚展示：随着样本逼近约束边界，代理模型的误差或可行性误判率显著上升。该图用于第三章问题定义与 failure mode 分析，是证明本文方法必要性的第一张关键实验图。

## Figure 3: Uncertainty Validation / Calibration

第三张图用于说明不确定性不是摆设。可以采用校准前后 coverage 对比图，或不确定性分位数—误差关系图，展示经过校准后高不确定性区域更一致地对应高误差区域。该图用于支撑“uncertainty 可作为可靠决策依据”的前提。

## Figure 4: Main Optimization Performance

第四张图是主结果图，展示在统一预算下，不同方法达到目标 spec 所需的仿真次数或随迭代推进的最佳性能变化。该图的目标不是单纯展示本文方法更快，而是突出它在可靠前提下节省仿真预算的能力。

## Figure 5: Optimization Trajectory Comparison

第五张图用于展示后期稳定性。可以选择若干代表性 runs，对比 surrogate-only、BO 和本文方法在后期的轨迹行为，例如最优值随迭代变化、是否反复触碰边界、是否出现多次误判后回退。该图的目的，是把“后期失效”从统计量转化为直观动态行为。

## Figure 6: Risk–Decision Visualization

第六张图用于说明本文的方法机制确实在起作用。可以展示高误判风险区域中系统更频繁地调用真实仿真，而在低风险区域更多依赖代理模型。这种图对解释 scheduler 和 decision logger 尤其重要，可大幅提升方法的可解释性。

## Figure 7: Hard Case Comparison

第七张图用于困难场景实验，重点展示在紧约束或冲突设置下，各方法的成功率、仿真预算消耗和误判事件频率差异。该图是“杀手级证据”，用来说明本文方法不仅平均表现更好，而且在最关键的 failure-amplified 场景下更可靠。

## Figure 8: Wall-Clock Analysis

第八张图展示不同方法的总时间分解，包括仿真时间、训练时间和推理时间。该图用于讨论工程意义，帮助读者理解本文方法的实际成本结构。

## Tables

表格方面，至少应包含一张主结果表，汇总不同方法在主实验上的仿真次数、成功率、最终性能与校准指标；一张消融表，展示去掉校准、utility 和风险建模后的退化情况；一张困难场景表，用于汇总 hard cases 下的关键结果。所有图表都应在结果目录中由脚本自动生成，且与具体配置版本保持对应关系。
